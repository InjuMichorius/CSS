Valid

This example uses the :valid and :unvalid property. The disadvantage is that the input immediately indicates invalid when the user is typing. Only when the user has typed the correct answer it will be seen as valid.

required

When we make the input required, the input will be invalid by default (because no answer is also invalid).

Not focus

This example uses the :valid:not(:focus) property. The invalid will not work when the input is focused, but instead be white. When the user clicks away from the input, the valid or invalid property will work.

Error message

It is not possbile to use before and after on an input element. The solution to this is using a span, and show this when the input is invalid.
